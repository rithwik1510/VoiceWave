# Architecture Decision Records

## Purpose

ADRs capture architecture-impacting decisions with rationale and consequences.

## Format

Use the Nygard ADR style in `docs/adr/template.md`.

## Naming

1. File pattern: `NNNN-short-title.md` (e.g., `0002-inference-worker-contract.md`)
2. Increment number sequentially.

## Required Sections

1. Status
2. Context
3. Decision
4. Consequences

## Workflow

1. Draft ADR in PR where decision is introduced.
2. Link ADR in PR template risk section.
3. Mark status (`Proposed`, `Accepted`, `Superseded`) during review.
